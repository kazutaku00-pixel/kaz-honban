# MVP要件定義書 — Language Lesson Platform（日本語オンラインレッスン）

---

## 1. プロダクト概要

### 1.1 ビジョン
「世界中の日本語学習者が、質の高い日本語講師と簡単につながれるプラットフォーム」

### 1.2 MVPの目的
PMF（Product-Market Fit）検証のための最小限プロダクト。

| 検証項目 | 検証方法 |
|---------|---------|
| レッスン需要があるか | 生徒の予約率・リピート率 |
| 講師が集まるか | 講師登録数・稼働率 |
| 予約→レッスン完了の転換率 | ファネル分析 |
| 適切な価格帯 | 予約数と価格の相関 |

### 1.3 ターゲットユーザー

#### ペルソナ1: 生徒（Student）
- 25〜40歳、アニメ/仕事で日本語に興味
- 英語圏（US, UK, AU）またはアジア圏（PH, TW, KR）
- italki, Preply, NativeCamp 経験者 or 初心者
- 月$50〜$150の予算感

#### ペルソナ2: 講師（Teacher）
- 日本語ネイティブ or JLPT N1保持者
- 副業 or フリーランスで教えたい
- 25〜50歳、オンライン教育に関心

#### ペルソナ3: 管理者（Admin）
- プラットフォーム運営者
- 講師の品質管理・売上管理

## 2. 技術スタック

| レイヤー | 技術 | 選定理由 |
|---------|------|---------|
| Frontend | Next.js 14 (App Router) | SSR/SSG、SEO、日本語圏への最適化 |

### 3.4 レッスン予約

#### 予約フロー
```
講師プロフィール → カレンダーで空き確認 → 日時選択（30min/60min）
→ 確認画面（料金表示）→ 決済（Stripe）→ 予約確定
→ 双方にメール通知
```

#### レッスンステータス
```
[scheduled] → [in_progress] → [completed]
     ↓              ↓
[cancelled]   [no_show]
```

#### キャンセルポリシー
- 24時間前まで：全額返金
- 12時間前まで：50%返金
- 12時間以内：返金なし
- 講師都合キャンセル：常に全額返金 + 次回割引

### 4.3 インデックス
```sql
CREATE INDEX idx_lessons_student ON lessons(student_id, start_time DESC);
CREATE INDEX idx_lessons_teacher ON lessons(teacher_id, start_time DESC);
CREATE INDEX idx_lessons_status ON lessons(status, start_time);
CREATE INDEX idx_reviews_teacher ON reviews(teacher_id, created_at DESC);
CREATE INDEX idx_teacher_profiles_approved ON teacher_profiles(is_approved) WHERE is_approved = true;
CREATE INDEX idx_notifications_user ON notifications(user_id, is_read, created_at DESC);
```

### 4.4 RLS（Row Level Security）ポリシー
```
users:        自分のデータのみ読み書き可。Admin は全て読み書き可。
teacher_profiles: 誰でも読み取り可（承認済みのみ）。本人のみ書き込み可。
lessons:      当事者（student/teacher）のみ読み取り可。
reviews:      誰でも読み取り可。lesson の student のみ書き込み可。
payments:     当事者のみ読み取り可。システムのみ書き込み可。
notifications: 本人のみ読み取り可。
```

## 7. 収益モデル

### MVPの収益構造
```
生徒の支払い額 = 講師の時給 + プラットフォーム手数料（20%）

例: 講師が $15/h → 生徒は $18 支払い → 講師 $15、プラットフォーム $3
```

### 将来の収益源（MVP後）
- サブスクプラン（月額固定でN回レッスン）
- プレミアム講師フィーチャー（上位表示）
- 教材販売手数料

- Supabase Auth + JWT
- Row Level Security（全テーブル）
- HTTPS必須
- Stripe決済（PCI DSS準拠）
- 入力バリデーション（クライアント + サーバー）

### パフォーマンス
- 初期表示 < 2秒（LCP）
- API応答 < 500ms
- 同時接続100ユーザー想定（MVP）
- 画像はSupabase Storage + CDN
- 講師一覧はページネーション（20件/ページ）

### 可用性
- Vercel: 99.99% SLA
- Supabase: 99.9% SLA
- Daily.co: 99.99% SLA

### 国際化（MVP）
- UI言語：英語のみ（MVP）
- タイムゾーン：全タイムゾーン対応必須
- 通貨：USD固定（MVP）

## 9. 競合分析・差別化

### 競合
| サービス | 強み | 弱み |
|---------|------|------|
| italki | 講師数多い、多言語 | 手数料高い、UI古い |
| Preply | ブランド力 | 高価格帯、講師の質にばらつき |
| NativeCamp | 月額固定、講師10,000人 | フィリピン中心、日本語特化ではない |
| Cafetalk | 日本語に強い | UIが古い、マーケが弱い |

### MVP差別化ポイント
1. **日本語特化** — 日本語学習に最適化されたUX
2. **講師品質** — 全講師を審査・承認制に
3. **シンプルなUX** — 3クリックで予約完了
4. **適正価格** — 手数料20%（italki 30%より安い）
5. **講師コミュニティ** — 講師同士の交流・研修

### 講師獲得戦略（最重要課題）
1. **初期30人**：日本語教師コミュニティ（Twitter/X, Reddit r/teachinginjapan）で直接スカウト
2. **インセンティブ**：最初の3ヶ月は手数料0%
3. **紹介プログラム**：講師が講師を紹介 → 双方にボーナス
4. **日本語学校連携**：非常勤講師の副業先として提案

## 14. リスクと対策

| リスク | 影響度 | 対策 |
|-------|-------|------|
| 講師が集まらない | 致命的 | 初期は手数料0%、直接スカウト |
| ビデオ通話品質 | 高 | Daily.co（安定性重視）、接続テスト機能 |
| 決済トラブル | 高 | Stripe Webhook + 手動確認フロー |
| 不正レビュー | 中 | レッスン完了者のみレビュー可 |
| スケジュールのタイムゾーン問題 | 中 | 全てUTC保存、表示時に変換 |
| 講師のno-show | 中 | ペナルティ制度、評価への反映 |
